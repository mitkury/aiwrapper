# How to write proposals

Write proposals as markdown files in [docs/dev/proposals].

Before writing one, explore how the current code and docs work so the proposal is grounded in the existing architecture.

Do not add time estimates. Assume implementation can move quickly.

Expect review and follow-up questions before implementation starts.

Keep this directory for work that is still proposed. Once a proposal is implemented or superseded, move any lasting usage or architecture guidance into the normal docs and remove the proposal. Git history preserves the original design discussion.
